# 3.2 订单预览与提交

## 3.2.1 订单预览

| 项目 | 说明 |
| --- | --- |
| 接口路径 | `POST /api/orders/preview` |
| 鉴权 | 是 |
| Content-Type | `application/json` |

### 功能描述

用户提交订单前进行订单预览。系统根据购物车商品、配送方式、地址、自提点、秒杀、砍价、满减、平台优惠券、商家优惠券和积分，重新计算最终应付金额并校验商品及库存。

该接口不会创建订单、扣减库存、核销优惠券或扣减积分。每次预览成功后，后端会把本次实际采用的最终选择保存到 Redis，并返回有效期为 900 秒的 `previewToken`，供提交订单使用。

### 请求头

```http
Authorization: Bearer {token}
Content-Type: application/json
```

### 请求示例

```json
{
  "cartItemIds": [24],
  "deliveryType": 1,
  "addressId": 4001,
  "pickupPointId": null,
  "usePlatformCoupon": true,
  "platformCouponUserId": null,
  "useMerchantCoupon": true,
  "merchantCouponUserId": null,
  "usePoints": true,
  "usePointsAmount": null,
  "activityContext": {
    "seckillProductId": null,
    "bargainRecordId": null,
    "groupId": null,
    "promotionIds": []
  },
  "remark": "请尽快配送"
}
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `cartItemIds` | `Long[]` | 是 | 当前用户选中的购物车明细 ID，至少一项且必须属于同一商家 |
| `deliveryType` | `Integer` | 否 | 配送方式：`1` 配送、`2` 自提；默认 `1` |
| `addressId` | `Long` | 条件必填 | 配送时使用；为空时使用当前用户默认地址 |
| `pickupPointId` | `Long` | 条件必填 | 自提时必填，且必须属于当前订单商家 |
| `usePlatformCoupon` | `Boolean` | 否 | 是否使用平台券，默认 `true` |
| `platformCouponUserId` | `Long` | 否 | 指定 `coupon_user.id`；启用平台券且为空时自动选择最优平台券 |
| `useMerchantCoupon` | `Boolean` | 否 | 是否使用商家券，默认 `true` |
| `merchantCouponUserId` | `Long` | 否 | 指定 `merchant_coupon_user.id`；启用商家券且为空时自动选择最优商家券 |
| `usePoints` | `Boolean` | 否 | 是否使用积分，默认 `true` |
| `usePointsAmount` | `Integer` | 否 | 指定使用积分；`usePoints=true` 且为空时使用本单最大可用积分 |
| `activityContext` | `Object` | 否 | 活动选择上下文 |
| `activityContext.seckillProductId` | `Long` | 否 | 指定 `seckill_product.id` |
| `activityContext.bargainRecordId` | `Long` | 否 | 指定用户砍价记录 ID |
| `activityContext.groupId` | `Long` | 否 | 拼团 ID，当前作为活动上下文保存 |
| `activityContext.promotionIds` | `Long[]` | 否 | 指定满减等促销活动 ID |
| `remark` | `String` | 否 | 用户备注，最多 500 字符 |

### 自动选择规则

- `usePlatformCoupon=true` 且 `platformCouponUserId=null`：自动选择最优平台券。
- `usePlatformCoupon=false`：不使用平台券。
- `useMerchantCoupon=true` 且 `merchantCouponUserId=null`：自动选择当前商家的最优商家券。
- `useMerchantCoupon=false`：不使用商家券。
- `usePoints=true` 且 `usePointsAmount=null`：使用本单最大可用积分。
- `usePoints=false`：不使用积分。
- `activityContext` 为空，或者其中活动 ID 全部为空：自动选择当前可用的最优活动，包括秒杀、砍价和满减。
- 预览响应中的 `activityInfo` 返回服务端实际采用的活动，而不是简单回显请求参数。

### 积分规则

```text
100 积分 = 1 元
```

```text
amountBeforePoints = totalAmount
                   - seckillDiscount
                   - bargainDiscount
                   - promotionDiscount
                   - merchantCouponDiscount
                   - platformCouponDiscount

maxDeductionAmount = amountBeforePoints × 10%
maxUsablePoints = min(用户积分余额, maxDeductionAmount × 100)
```

例如优惠后金额为 `99.80` 元时，最多抵扣 `9.98` 元，即 `998` 积分。用户即使有 `4000` 积分，本单也只能使用 `998` 积分。

### 重要响应说明

- `previewToken` 与当前登录用户绑定，不能跨用户提交。
- `previewToken` 有效期为 `previewExpireSeconds`，当前为 900 秒。
- 每次重新预览都会生成独立令牌，多窗口不会互相覆盖。
- 响应不再返回重复字段 `itemAmount`。
- 响应不再返回内部字段 `promotionSnapshot`。
- `promotionDetail` 只有积分项返回 `usedPoints`。

#### 积分超过本单上限

```json
{
  "code": 40952,
  "message": "积分抵扣超过本单可用上限",
  "data": {
    "maxUsablePoints": 998,
    "requestedPoints": 1000,
    "maxDeductionAmount": 9.98
  }
}
```

#### 平台优惠券不可用

```json
{
  "code": 40913,
  "message": "平台优惠券不可用或不满足使用条件",
  "data": {
    "platformCouponUserId": 1,
    "minConsume": 100.00,
    "currentAmount": 80.00
  }
}
```

#### 商家优惠券不可用

```json
{
  "code": 40914,
  "message": "商家优惠券不可用或不满足使用条件",
  "data": {
    "merchantCouponUserId": 7001,
    "merchantId": 2001,

```

#### Redis 暂时不可用

```json
{
  "code": 50301,
  "message": "订单预览状态保存失败，请稍后重试",
  "data": null

```

### 处理说明

- 查询并校验当前用户的购物车商品。
- 校验商品状态、审核状态、SKU 和库存。
- 校验商品必须属于同一商家。
- 根据配送方式校验地址或自提点。
- 计算秒杀、砍价和满减优惠。
- 平台券与商家券可各使用一张并叠加。
- 在所有非积分优惠计算完成后，再计算积分 10% 上限。
- 将实际采用的商品、配送、优惠券、积分、活动和备注保存到 Redis。
- 预览仅试算，不修改库存、优惠券或积分。

## 3.2.2 提交订单

| 项目 | 说明 |
| --- | --- |
| 接口路径 | `POST /api/orders` |
| 鉴权 | 是 |
| Content-Type | `application/json` |
| 幂等要求 | 请求头 `X-Idempotent-Key` 必填 |

### 功能描述

用户确认订单后，使用订单预览返回的 `previewToken` 创建订单。提交接口不再接收完整的购物车、地址、优惠券、积分和活动参数，也不接收任何金额。

后端从 Redis 读取本次预览实际采用的最终选择，然后重新校验商品、库存、活动、优惠券和积分并重新计价。校验通过后创建订单、扣减库存、核销优惠券并预占积分。

### 请求头

```http
Authorization: Bearer {token}
Content-Type: application/json
X-Idempotent-Key: order-submit-202607030001
```

### 请求示例

```json
{
  "previewToken": "f84de3a249e24763a39df29b2819ec32"
}
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `previewToken` | `String` | 是 | 订单预览返回的令牌，最长 64 字符，当前有效期 900 秒 |

请求体不能完全为空，因为后端需要通过 `previewToken` 确定提交哪一次预览。除令牌外，不需要重复传递其他订单参数。

### 成功响应（200）

```json
{
  "code": 0,
  "message": "订单创建成功",
  "data": {
    "orderId": 5001,
    "orderSn": "ORD202607010001",
    "status": 0,
    "statusText": "WAIT_PAY",
    "totalAmount": 139.80,
    "seckillDiscount": 20.00,
    "bargainDiscount": 0.00,
    "promotionDiscount": 0.00,
    "merchantCouponDiscount": 15.00,
    "platformCouponDiscount": 5.00,
    "pointsDeduction": 9.98,
    "deliveryFee": 0.00,
    "payAmount": 89.82,
    "expireTime": "2026-07-01 16:05:27"
  },
  "timestamp": 1783047372728,
  "requestId": "req-xxx"
}
```

### 重复幂等响应（200）

同一用户使用相同 `X-Idempotent-Key` 重复提交时，返回第一次创建的订单结果：

```json
{
  "code": 0,
  "message": "订单创建成功",
  "data": {
    "orderId": 5001,
    "orderSn": "ORD202607010001",
    "status": 0,
    "statusText": "WAIT_PAY",
    "totalAmount": 139.80,
    "payAmount": 89.82,
    "expireTime": "2026-07-01 16:05:27",
    "idempotent": true
  }
}
```

### 失败响应

#### 预览令牌过期或不存在

```json
{
  "code": 40961,
  "message": "订单预览已过期，请重新预览",
  "data": {
    "previewToken": "f84de3a249e24763a39df29b2819ec32"
  }
}
```

#### Redis 暂时不可用

```json
{
  "code": 50301,
  "message": "订单预览状态读取失败，请稍后重试",
  "data": null
}
```

提交时仍可能返回库存不足、活动失效、优惠券不可用、积分余额不足或积分超过上限等业务错误，因为最终价格和资格必须以提交时的实时校验为准。

### 处理说明

- 优先根据 `userId + X-Idempotent-Key` 查询已创建订单；存在时直接返回原订单。
- 根据 `userId + previewToken` 从 Redis 读取预览选择，禁止跨用户使用令牌。
- 重新查询购物车商品，不信任 Redis 中的商品价格或库存。
- 重新校验配送地址或自提点。
- 重新校验秒杀、砍价、满减、平台券和商家券。
- 重新校验积分余额及优惠后金额 10% 的抵扣上限。
- 按最新有效数据重新计算订单金额。
- 创建订单主表、订单明细和优惠快照。
- 扣减普通库存和秒杀库存。
- 核销平台券与商家券。
- 扣减积分并写入积分流水。
- 删除已提交的购物车明细。
- 写入订单状态流水。
- 整个创建过程使用数据库事务，任一步失败时整体回滚。
- 待支付订单超时后恢复库存、优惠券和积分。

### 前端调用流程

```text
POST /api/cart/checkout-preview
        ↓ 进入订单确认页
POST /api/orders/preview
        ↓ 获取 previewToken
POST /api/orders
        ↓ 请求体只传 previewToken
支付订单
```
